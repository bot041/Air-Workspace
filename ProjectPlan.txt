
AIRSCRIPT MVP v2.2 - PROJECTPLAN.txt
====================================

PROJECT OVERVIEW
----------------
AirScript is a desktop AR-style gesture workspace that uses a webcam and computer vision to let users create, manipulate, organize, and transfer hand-drawn objects without touching a mouse, keyboard, or screen.

CORE VISION
-----------
- Live webcam feed visible at all times.
- Draw in the air using hand gestures.
- Preserve exactly what the user draws.
- No OCR.
- No handwriting recognition.
- No AI interpretation of drawings.
- Single-hand interaction.
- Gesture-first workflow.
- Workspace and Output Tray organization system.

PLATFORM STRATEGY
-----------------

GESTURE SYSTEM
--------------

1. DRAW
Gesture:
- Index Finger Only (☝️)

Action:
- Draw strokes in real-time.

2. NAVIGATION
Gesture:
- Index + Middle Finger (✌️)

Action:
- Move hand without drawing.
- Reposition hand safely.

3. SELECT
Gesture:
- Pinch (🤏)

Action:
- Select object.

4. DRAG
Gesture:
- Pinch + Move

Action:
- Drag selected object.

5. DROP
Gesture:
- Release Pinch

Action:
- Drop object at current position.

6. PARTIAL ERASE
Gesture:
- Open Palm Swipe (✋)

Action:
- Remove touched stroke segments.

7. DELETE OBJECT
Gesture:
- Open Palm Hold for 1 Second

Action:
- Delete entire object.

DRAWING RULES
-------------
- Fixed stroke thickness.
- Multi-stroke objects supported.
- Exact stroke preservation.
- Stroke smoothing enabled.
- Drawings can be:
  * Text
  * Signatures
  * Diagrams
  * Symbols
  * Sketches
  * Equations

NO OCR / RECOGNITION
--------------------
- No OCR.
- No handwriting recognition.
- No text classification.
- No symbol recognition.
- No AI conversion.

OBJECT CREATION
---------------
Workflow:

DRAWING
  ↓
CREATE STROKES
  ↓
1 SECOND INACTIVITY
  ↓
FINALIZE OBJECT
  ↓
IMMEDIATELY INTERACTIVE

MULTI-STROKE OBJECTS
--------------------
A single object may contain multiple strokes.

Example:
HELLO
- H = multiple strokes
- E = multiple strokes
- L = multiple strokes
- O = one stroke

All strokes belong to one object until:
- 1 second inactivity occurs.

OBJECT DATA STRUCTURE
---------------------

StrokeObject
{
    object_id
    strokes[]
    position
    bounding_box
    z_index
    selected
    panel_location
    timestamp
    theme
}

SELECTION SYSTEM
----------------
- Pinch inside object bounds.
- Top-most object gets selected.
- Selected object glows.
- No bounding box.
- No resize handles.

OBJECT OVERLAP RULES
--------------------
- Objects can overlap.
- Objects can stack.
- Top-most object wins selection.
- Dragged object temporarily moves to highest z-index.

DRAGGING BEHAVIOR
-----------------
- Natural grab offset.

Example:
Object position = (300, 200)
Pinch point = (340, 220)

Offset = (+40, +20)

During drag:
- Maintain offset.
- No snapping.
- No jumping.

ERASING SYSTEM
--------------

Partial Erase:
- Palm swipe removes touched stroke segments.

Full Delete:
- Palm hold for 1 second over object.
- Entire object deleted.

Visual Feedback:
- Red glow before deletion.
